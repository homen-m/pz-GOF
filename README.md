# Реалізація GOF Патернів Проєктування на TypeScript

## Як запустити проєкт

1. Встановіть залежності: `npm install`
2. Запустіть демонстраційний код: `npm start`

## Опис реалізованих патернів

### Породжувальні патерни (Creational)
1. **Builder (Voxel Game Structures)**
  * **Проблема:** Ініціалізація ігрових структур з великою кількістю опціональних параметрів.
  * **Рішення:** Покрокова конфігурація блоків, матеріалів та фурнітури через зрозумілий ланцюжок викликів.
2. **Singleton (Distributed Computing Coordinator)**
  * **Проблема:** Уникнення конфліктів диспетчеризації завдань у кластері.
  * **Рішення:** Заборона створення кількох інстансів координатора. Реєстрація вузлів відбувається централізовано.

### Структурні патерни (Structural)
3. **Adapter (Learning Platforms Integration)**
  * **Проблема:** Несумісність форматів даних власного дашборду з API сторонніх освітніх платформ (наприклад, JavaRush).
  * **Рішення:** Адаптер конвертує JSON-відповіді зовнішнього API у простий масив рядків, який очікує наша система.
4. **Facade (Data Science Pipeline)**
  * **Проблема:** Складна ініціалізація підсистем для машинного навчання та аналізу (завантаження, парсинг, побудова моделей).
  * **Рішення:** Єдиний метод `runAnalysis()`, який ховає під капотом виклики всіх необхідних бібліотек у правильному порядку.

### Поведінкові патерни (Behavioral)
5. **State (Jedi Combat Stances)**
  * **Проблема:** Зміна правил розрахунку шкоди та анімацій залежно від екіпірованої зброї персонажа.
  * **Рішення:** Патерн "Стан" переносить логіку стійок в окремі класи. Метод атаки просто делегує виконання активному стану.
6. **Strategy (Racing Car Setup)**
  * **Проблема:** Необхідність перемикати фізичну модель управління автомобілем на льоту (Гріп або Дрифт).
  * **Рішення:** Фізика входження в поворот інкапсульована у змінні стратегії, що дозволяє уникнути `if/else` залежностей від поточного тюнінгу.
